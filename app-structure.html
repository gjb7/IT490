Files

app/

app/ holds all the files specific to your application. That is, any custom code that you might have to write to get your app working. This includes specifying the routes for your application, the models, controllers, and views your app uses, and the database migrations for your app.

config.php

config.php holds basic information to configure your application. Configuration options include setting the app in debug mode or setting up the view system.

controller/

controller/ is where you put all the classes that are executed as a the result of URL being visited. They will process any input, and send back some kind of output to the browser.

database.php

database.php is the file where you put your database info. This project has only been tested with MySQL.

migration/

migration/ is where all database migrations (schema changes for the database) are stored. For information on creating and using migrations, see the migration docs.

model/

model/ holds all the model classes for the app. This project uses Eloquent for its model and database implementations, with a few changes. You should inherit from IT490\Model instead of Eloquent, as we implement some custom validation techniques.

routes.php

routes.php specifies the routes for your application. See the docs for the Router class for details on methods you can use to specify routes.

views/

views/ holds all the view templates used by the system. Templates are written in Twig syntax. For information on creating templates, see the templates docs.
